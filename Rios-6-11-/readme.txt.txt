1) CI/CD es una mejor práctica para maximizar el uso de métodos ágiles. los beneficios que tenemos al usarlo son que nos permiten que los equipos de desarrollo de software se concentren en cumplir con los requisitos comerciales, la calidad del código que se está desarrollando y la seguridad del proceso. 

2)El ambiente de pruebas implementa una versión estable de un sistema o aplicación para que los equipos de prueba puedan realizar controles de calidad y pruebas de seguridad relevantes en este entorno para verificar que no haya vulnerabilidades que comprometan su integridad y disponibilidad, así como para verificar que sus mitigaciones afectan el sistema o influencia de la aplicación.
  
 El ambiente de produccion es donde el usuario final consume el servicio. Es aquí donde se ajustan los elementos necesarios para mantener la disponibilidad del servicio y se programan los respaldos del entorno.

El ambiente de staging se utilizan para probar el código, compilar y actualizar en producción para garantizar la calidad antes de la implementación de la aplicación.
